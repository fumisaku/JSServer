GMフリーズ修正レポート

対象ファイル: F_GM.vb　／　コミット: d40fd2c　／　ブランチ: master

1. 不具合概要

症状
全ジャッジがSENDを完了してGM画面の全ジャッジセルが水色になった後、画面がフリーズし、 F501の採点集計が更新されず、関連端末へのデータ送信も行われなかった。

発生条件

条件内容
電文種別SNDRESLT_V2_J（ジャッジ端末からの採点結果SEND）
必須状態Timer1（競技タイマー）が動作中（TimerStatus = "P" / "D" / "J"）
発生タイミング最後のジャッジのSEND受信と Timer1_Tick（1秒周期）が同時に重なった瞬間
頻度ごくまれに（タイミング依存のレースコンディション）

2. 根本原因

UIスレッド上からの二重 Invoke によるデッドロック

SNDRESLT_V2_J 受信時、非同期スレッドが Me.Invoke(Sub()...J_Send色付け()...End Sub) でUIスレッドに処理を委譲する。 同時に Timer1_Tick が Me.Invoke(TimerDelegate,...) を投入してUIスレッドの完了を待っている。 この状態で全SEND確定後に Stopタイマー() が呼ばれると、内部でさらに Me.Invoke(TimerDelegate,...) を実行し、 UIスレッドがUIスレッド自身の空きを待つデッドロックが発生した。 

【デッドロック発生の構造】

[Timer1_Tick スレッド]
  Me.Invoke(TimerDelegate, ...)        ← UIスレッドに Invoke 投入・完了待ち中

[非同期受信スレッド]（最後のSENDを受信）
  Me.Invoke(Sub()                      ← UIスレッドの完了待ちでブロック
    J_Send色付け()
      全SENDFLAG = True
        Stopタイマー()
          Me.Invoke(TimerDelegate)     ← UIスレッドの空きを待つ → 永久ブロック
  End Sub)

[UIスレッド]
  Timer1_Tick の TimerDelegate を実行中
  → 終わったら非同期受信の Sub を実行したいが
     その Sub の中からさらに Invoke が来るのを待っている
  → 完全にデッドロック

3. 修正内容

修正① Stopタイマー() 案A

F_GM.vb 行 2502〜2526　／　InvokeRequired による分岐を追加。 非同期スレッドから呼ばれた場合のみ Invoke を実行し、UIスレッド上では直接 Timer更新() を呼ぶ。 

 Private Sub Stopタイマー()
-    Me.Invoke(TimerDelegate, New Object() {"  00:00", SystemColors.Control})
+    ' 非同期スレッドから呼ばれた場合のみ Invoke する（UIスレッド上からの呼び出しだと二重Invokeでデッドロックになるため）
+    If Me.InvokeRequired Then
+        Me.Invoke(New Action(AddressOf Stopタイマー))
+        Return
+    End If
+    Timer更新("  00:00", SystemColors.Control)
 
     TimerStatus = "N"
     Send_TimStart電文("N", "")
     SEND_JK_TIMER("N", "")
     If Movie_Status = "ON" Then

 End Sub

修正② 更新タイマーStart() 案A

F_GM.vb 行 1203〜1212　／　同様に InvokeRequired による分岐を追加。 UIスレッド上では直接 Timer2Start() を呼ぶ。 

 Private Sub 更新タイマーStart()
     更新タイマー実施中FLAG = True
-    Me.Invoke(Timer2_Start_Delegate, New Object() {})
+    ' 非同期スレッドから呼ばれた場合のみ Invoke する（UIスレッド上からの呼び出しだと二重Invokeでデッドロックになるため）
+    If Me.InvokeRequired Then
+        Me.Invoke(Timer2_Start_Delegate, New Object() {})

+    End If
 End Sub

修正③ Thread.Sleep(1000) の削除 案C

F_GM.vb 行 1156　／　全SEND確定後のUIスレッド上での不必要な1秒待機を削除。

4. 既存動作への影響

呼び出し経路修正前修正後
REQTIMSTART（非同期スレッド）から Stopタイマー() InvokeRequired=True → Invoke 実行 同じ（変化なし）
全SEND検知（UIスレッド上）から Stopタイマー() 二重 Invoke → デッドロック InvokeRequired=False → 直接呼出し（修正済み）
タイマーボタン押下（UIスレッド）から Stopタイマー() 二重 Invoke の可能性 直接呼出し（安全）
REQTIMSTART（非同期スレッド）から 更新タイマーStart() InvokeRequired=True → Invoke 実行 同じ（変化なし）
全SEND検知（UIスレッド上）から 更新タイマーStart() 二重 Invoke → デッドロック InvokeRequired=False → Timer2Start() 直接呼出し（修正済み）
